Welcome to Iris

Iris is a statically-typed backend language that runs on a dedicated VM. Designed for expressive error handling, declarative routing, and clean generics — without the noise.

// designed for the backend

Expressive. Safe. Elegant.

Iris combines a powerful type system with first-class error handling, metatype-driven reflection, and a clean syntax inspired by the best of modern languages — without their baggage.

Quick Start → Install
⚡
VM-based Runtime
Runs on a dedicated bytecode VM. Fast startup, portable, and inspectable.
🔒
Typed Error Handling
First-class ! and ? type modifiers. No exceptions, no panics.
🏷️
Metatypes
Attach typed metadata to functions. Access it at runtime with zero magic.
🧩
Generics & Traits
Parametric types and trait-based dispatch. Clean, no lifetime annotations.
🔁
Pattern Matching
Full match with ranges, nullability, and exhaustive checks.
📦
Module System
Scoped imports via std::{ io, http }. Clean namespacing by default.

A first look

iris
import std::{ io, http };

@Metatype
type Get {
    path: String
}

@Get(path = "/user")
fn getUser() -> HttpResponse? {
    // Return a response or Null
}

error MathError { DivisionByZero }

fn divide(a: Float, b: Float) -> Float! {
    if b == 0 {
        return .DivisionByZero;
    }
    return a / b;
}

fn main() -> Void! {
    let result = divide(10, 0) catch -1;
    try io.println(result);
}

Installation

Install the Iris VM and toolchain on your machine. Currently supports Linux and macOS.

⚠
Iris is currently in alpha. The language specification is not yet stable. Use in production at your own risk.

via curl (recommended)

shell
curl -fsSL https://iris-lang.dev/install.sh | sh

Manual build

shell
git clone https://github.com/iris-lang/iris
cd iris
make build
export PATH="$PATH:$(pwd)/bin"

Verify installation

shell
iris --version
# iris 0.1.0-alpha (vm build 2024-01)
💡
VS Code extension with syntax highlighting is available at ext install iris-lang.iris-vscode.

Quick Start

Build and run your first Iris program in under 5 minutes.

Create a project

shell
iris new my-app
cd my-app

This creates the following structure:

tree
my-app/
├── iris.toml        # project manifest
├── src/
│   └── main.iris   # entry point
└── .gitignore

Hello, World

iris · src/main.iris
import std::io;

fn main() -> Void! {
    try io.println("Hello, World!");
}
shell
iris run src/main.iris
# Hello, World!

A minimal HTTP server

iris
import std::{ io, http };

@Metatype
type Get { path: String }

@Get(path = "/ping")
fn ping() -> HttpResponse? {
    return http.ok("pong");
}

fn main() -> Void! {
    let server = try http.serve(port: 8080);
    try io.println("Listening on :8080");
    try server.run();
}

Types & Variables

Iris is statically typed with local type inference. Every variable has a known type at compile time.

Primitive types

TypeDescriptionExample
Int64-bit signed integerlet x: Int = 42
Float64-bit floatlet f: Float = 3.14
BoolBooleanlet b: Bool = true
StringUTF-8 stringlet s: String = "hi"
VoidNo return valuefn f() -> Void
NullAbsence of valuelet n = Null

Type modifiers

Iris uses postfix modifiers to express nullability and fallibility at the type level:

T — always present
T? — nullable
T! — may error
T!? — nullable AND may error
iris
fn alwaysInt()   -> Int    { return 1; }
fn maybeInt()    -> Int?   { return Null; }
fn riskyInt()    -> Int!   { return .SomeError; }
fn bothInt()     -> Int!?  { return Null; }

Variable declaration

iris
// Explicit type
let x: Int = 42;

// Inferred type
let y = 42;           // Int
let name = "Iris";    // String
let nums = [1, 2, 3]; // []Int

// Explicit array type
let ids: []Int = [1, 2, 3];
ℹ
All variables in Iris are immutable by default. Mutability modifiers are planned for a future release.

Functions

Functions are first-class citizens in Iris. They support generics, typed returns, and can carry metadata via metatypes.

Basic syntax

iris
fn add(a: Int, b: Int) -> Int {
    return a + b;
}

// Nullable return
fn findUser(id: Int) -> User? {
    // ...
}

// Fallible return
fn parseJson(raw: String) -> Json! {
    // ...
}

Functions as values

iris
// Function type annotation
let handler: fn() -> HttpResponse? = getUser;

// Passing functions as arguments
fn applyTwice(f: fn(Int) -> Int, x: Int) -> Int {
    return f(f(x));
}

Named parameters

iris
fn connect(host: String, port: Int) -> Conn! { ... }

// Call with named params
let conn = try connect(host: "localhost", port: 5432);
// language

Error Handling

Iris handles errors at the type level. No exceptions, no panics. Errors are declared, returned, and handled explicitly.

Declaring errors

iris
error MathError {
    DivisionByZero,
    Overflow
}

error DatabaseError {
    ConnectionFailed,
    QueryFailed,
    NotFound
}

Returning errors

Use dot-prefix shorthand to return an error variant:

iris
fn divide(a: Float, b: Float) -> Float! {
    if b == 0 {
        return .DivisionByZero;  // shorthand for MathError.DivisionByZero
    }
    return a / b;
}

Handling errors

catch — provide a fallback value

iris
let result = divide(10.0, 0.0) catch -1.0;

try — propagate error to caller

iris
fn compute() -> Float! {
    let x = try divide(10.0, 2.0);  // propagates on error
    return x * 2.0;
}

match — exhaustive error handling

iris
let val = match divide(10.0, 0.0) {
    Ok(v)               => v,
    Err(.DivisionByZero) => 0.0,
    Err(.Overflow)       => Float.MAX
};

The T!? type

A function returning T!? can produce a value, Null, or an error — all handled distinctly:

TypeSemanticsResult Set
TAlways returns T{value}
T?May be absent{value, Null}
T!May fail{value, error}
T!?May be absent OR fail{value, Null, error}

Pattern Matching

The match expression in Iris is exhaustive, supports ranges, and works seamlessly with nullable and error types.

Basic match

iris
let label = match score {
    90..100 => "A",
    80..89  => "B",
    70..79  => "C",
    _       => "F"
};

match is an expression

Match arms are expressions — no return needed inside arms:

iris
return match a {
    1    => Null,
    1..9 => self.get(a),
    _    => .SomeError
};
⚠
Do not use return inside match arms — the match expression itself is the value. Using return inside arms is a syntax error in Iris.

Matching nullable

iris
let user: User? = findUser(42);

match user {
    Null   => try io.println("not found"),
    u      => try io.println(u.name)
};

Types & Structs

Iris uses type to declare structured data. Methods are defined separately via impl.

Declaring a type

iris
type User {
    id:    Int,
    name:  String,
    email: String?
}

Implementing methods

iris
impl User {
    fn new(id: Int, name: String) -> Self {
        return Self {
            id:    id,
            name:  name,
            email: Null
        };
    }

    fn greet(self) -> String {
        return "Hello, " + self.name;
    }
}

let u = User::new(1, "Alice");
try io.println(u.greet());

Instantiation

iris
let u = User {
    id:    42,
    name:  "Bob",
    email: "bob@example.com"
};

Generics

Iris supports parametric types on both type declarations and fn definitions. No variance annotations, no wildcards.

Generic types

iris
type Pair[A, B] {
    first:  A,
    second: B
}

let p = Pair[Int, String] { first: 1, second: "one" };

Generic impl

iris
type HashMap[K, V] {
    keys:   []K,
    values: []V
}

impl[K, V] HashMap[K, V] {
    fn new() -> Self {
        return Self { keys: [], values: [] };
    }

    fn get(self, key: K) -> V? {
        // lookup logic
    }
}

Generic functions

iris
fn first[T](list: []T) -> T? {
    if list.len == 0 { return Null; }
    return list[0];
}
// language

Traits

Traits define shared behaviour. Any type can implement any trait via impl Trait for Type.

Defining a trait

iris
trait Display {
    fn toString(self) -> String;
}

trait Index[K, V] {
    fn indexOf(self, index: K) -> V;
}

Implementing a trait

iris
impl Display for User {
    fn toString(self) -> String {
        return "User(" + self.name + ")";
    }
}

// Generic trait impl
impl[K, V] Index[K, V?] for HashMap[K, V] {
    fn indexOf(self, index: K) -> V? {
        return self.get(index);
    }
}

Trait bounds

iris
fn print[T: Display](val: T) -> Void! {
    try io.println(val.toString());
}

Metatypes

Metatypes let you attach typed, structured metadata to functions. Access it at runtime — no reflection magic required.

✦
Metatypes are one of Iris's defining features. They enable declarative routing, middleware, and tooling — all typesafe.

Declaring a metatype

iris
@Metatype
type Get {
    path: String
}

@Metatype
type Middleware {
    name:     String,
    priority: Int
}

Annotating functions

iris
@Get(path = "/user")
fn getUser() -> HttpResponse? {
    // handler logic
}

@Get(path = "/posts")
@Middleware(name = "auth", priority = 1)
fn getPosts() -> HttpResponse? {
    // handler logic
}

Reading metadata at runtime

iris
let routes: HashMap[String, fn() -> HttpResponse?] = HashMap::new();

fn register(handler: fn() -> HttpResponse?) -> Void {
    let meta: Immutable[Get]? = handler.meta[Get];

    if meta != Null {
        routes[meta.path] = handler;
    }
}

register(getUser);
register(getPosts);

The Immutable wrapper

Metadata returned via .meta[T] is wrapped in Immutable[T] to prevent accidental mutation of compile-time annotations. Access fields normally, but reassignment is forbidden.

io.writeFilefn(path: String, data: String) -> Void!Write to file

fn main() -> Void! {
    try io.println("Enter your name:");
